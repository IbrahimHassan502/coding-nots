google fonts in wordpress

the problem with adding google fonts is that two styles have rel='preconnect' and one of them even have an attribute crossorigin while the default when enqueueing a style is rel='stylesheet' so I used this piece of code I found on apply_filters() docs from a user contribute notes 

                add_filter( 'style_loader_tag', 'wpdocs_remove_https_styles', 10, 2 );
                function wpdocs_remove_https_styles( $html, $handle ) {
                  $handles = array( 'twentysixteen-fonts', 'open-sans' );
                  if ( in_array( $handle, $handles ) ) {
                    $html = str_replace( 'https:', '', $html );
                  }
                  return $html;
                }

#10 - Tips to Secure Your WordPress Website

before securing wordpress you have to secure your machine (he won't talk about securing your machine he will talk about wordpress)

quick tips to secure your wordpress site:

  • first way is to secure the dashboard: one way to get to the dashboard is to type in the link of your wordpress site (either locally so you go to localhost/wordpress or hosted so you type the url to the domain you purchased) followed by /wp-admin

    you can make a firewall on any folder you have on your website (including the wp-admin) to make a firewall:

    1. elzero went to direct admin
    2. chose password protected directory
    3. chose find a directory to password protect
    4. chose wordpress
    5. chose wp-admin
    6. in the protected directory prompt you type the message you want it browsed on the firewall (like staff only or whatever)
    7. type in the user who can access through the fire wall
    8. type in the password
    9. check the protection enable checkbox before the save button
    10. hit save

    now when you try to access the dashboard you'll be asked to enter the username and password of the fire wall (to be able after that to enter the username and password to the actual site)

    now notice if you type the website link followed by /wp-login.php it will open the dashboard login page just fine without authintication or firewall

    to solve the previous problem you can add a rename plugin to your website that will rename the wp-login.php for you so the hacker can't access it

    in the video elzero just googled wp-login rename and said that you can find the plugin that suites you best

  • another tip is to always make sure to update the wordpress version (from the dashboard) as they continously patch loopholes

    also you can enable auto update but be aware is it may break some stuff down in some customized wordpress versions

    just google wp auto update and you will find some codes to add to the wp-config file (he illustrated how to auto update I just don't have energy to type)

  • only use the absolute neccessary plugins don't take the easy route and make any small thing using plugin just because you don't want to search how it's done
  • don't use cracked or nulled plugins as it will probably contain viruses

  • make sure your password to access the site is hard to crack

    you can use a password generator like identity safe from norton

    and with such hard randomly generated password you might want to use a passowrd saving application like roboform or you can just count on google password

  • you can use a plugin to make a two step verfication or 2fa so when someone tries to access the website it won't let him (with the right username and password) it won't let him untill you verfy the login from your phone
  • make sure you're with a good hosting company so the hacker doesn't hack you from the server
  • make sure you're using the right file permissions (he illustrated them but I didn't like to type them)

#040 - WP Enqueue Style Function

in this lesson he explains how to add your style and script files and they're added on two steps

  • making a custom function which uses wp function to add the style/script
  • calling this function using what's called a hook or an action

in this lesson he only expalins the first part which is creating a custom function with the wp function in it

the following functions go in functions.php

  • creating a custom function is exaxtly like in js function add-my-custom-styles() {....}
  • to include the style file we use a wp function called wp_enqueue_style($handle, $src, $dependancies or $deps, $version or $ver, $media)
    • $handle: the name of the stylesheet you're including (and according to the docs it must be unique) but it doesn't have to be the same name on the file so you can be including a file with a name bootstrap.min.css but you give it a name of bootstrap-style
    • $src: the source of the file you're including and it consists of two parts
      • the path up to the theme folder and this one you get using get_template_directory_uri()
      • the path of the file from that point so it can be somethign like that /css/bootstrap.min.css
      • and you type the two by concatination so you should type something like that in $src paramater get_template_directory_uri() . '/css/bootstrap.min.css'
    • $deps: An array of registered stylesheet handles this stylesheet depends on.
    • $ver: stylesheet version number, if it has one, which is added to the URL as a query string so it is recognized as a new version in case of caching

      If version is set to false, a version number is automatically added equal to current installed WordPress version. If set to null, no version is added.

    • $media: just like css media it defines The media for which this stylesheet has been defined.

      the default value is all but it can be print, screen, or (max-width: 640px) just like the css

    just the handle and src are the required params the others are optional

so the end resault should look like this

                function add-styles() {
                wp_enqueue_style('main-style', get_template_directory_uri(). '/public/output.css');
                }

to add another style you use wp_enqueue_style() again within the custom function

#043 - WP_Enqueue vs WP_Register

(as much as I understood) it's a function used to load a script: wp_register_script() or a style wp_register_style() to make them available (like a variable) so in case you needed to enqueue them multiple times wordpress doens't have to look for them every time it just gets the loaded (registered version)

just like in js when you create a loop depeneding on the array's length you but the length in a variable so the interpreter doesn't calculate the length every time the code in the loop runs

also registered scripts/styles are recognized just by the name so (since jquery is registered by wp by default) this code will work just fine wp_enqueue_script('jquery') otherwise (in case it wasn't registered) it shouldn't be working (I didn't really test it)

also registered scripts can be used as $deps like wp_enqueue_script('bootstrap-js', get_template_directory_uri(). '/js/bootstrap.js', array('jquery'), false, true) note that you just type the name of the script (just jquery) and it will be loaded
